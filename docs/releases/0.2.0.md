# pyspharm-ng 0.2.0

## Summary

`pyspharm-ng 0.2.0` expands the maintained `pyspharm` interface from core
scalar and wind transforms to common scientific diagnostics, spectral
regridding, grid metadata and point interpolation. It also restores the bundled
scientific examples as reproducible command-line programs.

## Highlights

- Diagnose scalar gradients with `SphericalHarmonicTransform.gradient`.
- Diagnose streamfunction and velocity potential from horizontal wind with
  `SphericalHarmonicTransform.streamfunction_velocity_potential`.
- Spectrally regrid scalar fields through `pyspharm.regrid_scalar`.
- Access Gaussian coordinates and weights, triangular spectral indices and
  icosahedral geodesic points through the maintained API.
- Interpolate a compact scalar spectrum at a latitude/longitude point with
  `pyspharm.interpolate_scalar`.
- Run documented examples for regridding, spherical harmonics, geodesic points
  and the Galewsky shallow-water test case without Basemap or interactive input.

## Compatibility

- Supported Python versions: CPython 3.10, 3.11 and 3.12.
- Supported binary-wheel platform: Linux x86_64 with manylinux-compatible
  glibc.
- Source distributions remain available for compatible C and Fortran compiler
  environments.
- `pyspharm` is the maintained public interface.
- `spharm` and `spharm.Spharmt` remain supported compatibility interfaces.

## Numerical contract

No SPHEREPACK routine or generated Fortran kernel was changed for this release.
The new public operations delegate to the established compatibility engine and
are validated against direct legacy calls. `legacy-contract-v1` continues to
pass for regular and Gaussian grids with stored and computed Legendre
workspaces.

The transform ABI remains explicit: gridded data use `float32` and compact
spectra use `complex64`. Gaussian latitude/weight metadata intentionally uses
`float64`, matching the historical helper.

## Upgrade notes

New applications can use the maintained interfaces directly:

```python
import pyspharm

source = pyspharm.SphericalHarmonicTransform(144, 72, grid="gaussian")
destination = pyspharm.SphericalHarmonicTransform(128, 65, grid="regular")
field = pyspharm.as_real32(field_from_application)
regridded = pyspharm.regrid_scalar(source, destination, field)

coefficients = destination.analyze_scalar(regridded, truncation=42)
dlon, dlat = destination.gradient(coefficients)
value = pyspharm.interpolate_scalar(
    coefficients,
    latitude=-23.55,
    longitude=313.0,
)
```

Existing programs using `spharm` remain compatible. The examples directory
contains runnable scripts and the Stage 7 guide documents the public API
boundary.

## Provenance

Release artifacts are produced only after an annotated `v0.2.0` tag is pushed
from the reviewed release commit. The GitHub Release workflow builds and tests
source and manylinux wheel artifacts. PyPI publication remains deliberately
outside this workflow.
