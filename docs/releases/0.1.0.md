# pyspharm-ng 0.1.0

## Summary

`pyspharm-ng 0.1.0` establishes a maintained distribution and compatibility
baseline for the historical Python interface to NCAR/UCAR SPHEREPACK.

## Highlights

- The maintained public API is available through `import pyspharm`.
- Existing applications can continue to use `import spharm` and `Spharmt`.
- Scalar and wind transforms are exposed through
  `SphericalHarmonicTransform` with explicit native precision boundaries.
- The build uses Meson/PEP 517 and no longer relies on `numpy.distutils`.
- The legacy Fortran sources are converted to free form during the build while
  retaining their original implementation as provenance input.
- The release includes source distributions and verified manylinux x86_64
  wheels for CPython 3.10, 3.11 and 3.12.

## Compatibility

- Supported Python versions: CPython 3.10, 3.11 and 3.12.
- Supported binary-wheel platform: Linux x86_64 with manylinux-compatible
  glibc.
- Other platforms can build from the source distribution with a compatible C
  compiler and `gfortran`.
- `pyspharm` is the maintained API.
- `spharm` and `spharm.Spharmt` remain supported compatibility interfaces.

## Numerical contract

The release passes `legacy-contract-v1` across regular and Gaussian grids with
stored and computed Legendre workspaces. No intended change to numerical
output, coordinate conventions or the retained single-precision SPHEREPACK ABI
is included in this release.

## Upgrade notes

Use the new API for new code:

```python
import pyspharm

transform = pyspharm.SphericalHarmonicTransform(144, 72, grid="gaussian")
field = pyspharm.as_real32(field_from_application)
coefficients = transform.analyze_scalar(field, truncation=42)
```

Existing `spharm` programs remain compatible. See the Stage 4 API guide for
method mappings and precision requirements.

## Provenance

Release artifacts are built from Git tag `v0.1.0` by the verified release
workflow. PyPI publication is deliberately not automated by this release.
